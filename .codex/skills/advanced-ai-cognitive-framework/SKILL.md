# Advanced AI Cognitive Framework

## 目標

本技能不是單純解釋 AI，而是把問題轉換成：

1. AI 能力層級判讀
2. 失敗模式診斷
3. 證據與驗證設計
4. 工具與資料來源配置
5. 人機責任邊界
6. 可執行的下一步

核心原則：

```text
生成不等於事實
流暢不等於理解
推理不等於驗證
工具不等於治理
自動化不等於可交付
```

## 何時啟動

遇到以下任務時主動使用：

- 分析 AI 回答為何出錯
- 設計 AI 課程、教材或能力地圖
- 評估 Prompt、Context、RAG、Agent 或 MCP 架構
- 建立企業 AI 導入成熟度模型
- 審查 AI 工作流的風險、品質與治理
- 將抽象 AI 概念轉成 SOP、規格或 GitHub 文件
- 使用者要求「不要討好、要驗證、要落地」

## 十二層認知模型

| 層級 | 模組 | 核心問題 | 主要風險 | 改善手段 |
|---|---|---|---|---|
| L1 | Probability | 模型如何生成？ | 高機率錯誤仍可被輸出 | 限定任務、要求證據 |
| L2 | Pattern | 模型如何學習？ | 模式被誤當因果 | 案例對照、反例測試 |
| L3 | Context | 模型看見什麼？ | 缺資料、舊資料、衝突資料 | 上下文盤點 |
| L4 | Prompt | 任務如何定義？ | 模糊需求導致錯方向 | 明確角色、目標、限制、格式 |
| L5 | Reasoning | 問題如何拆解？ | 推理跳步、隱藏假設 | 分解、反證、檢查點 |
| L6 | Tools | 如何取得與操作外部世界？ | 工具選錯、結果未查核 | 工具路由、權限控制 |
| L7 | Memory | 如何延續個人與專案脈絡？ | 過期記憶、錯誤記憶 | 記憶分層、更新規則 |
| L8 | Retrieval | 如何取得專屬或最新知識？ | 檢索錯誤、來源品質差 | RAG、來源排序、引用 |
| L9 | Agent | 如何規劃與執行多步任務？ | 目標漂移、無限迴圈 | 狀態機、預算、停止條件 |
| L10 | Human-AI | 人與 AI 如何分工？ | 責任真空、自動化偏誤 | 人類核准點、責任矩陣 |
| L11 | Governance | 如何可信、安全、合規？ | 隱私、偏誤、版權、注入攻擊 | 稽核、紅隊、政策控制 |
| L12 | AI OS | 如何形成可持續系統？ | 工具堆疊但無營運閉環 | 指標、流程、治理、持續改善 |

## 標準執行流程

### Step 1：任務分類

先判斷任務屬於哪一類：

- 認知解釋
- 內容生成
- 決策分析
- 研究查證
- 系統設計
- 程式執行
- 高風險專業建議

### Step 2：風險分級

```text
低風險：腦力激盪、初稿、格式整理
中風險：商業判斷、課程內容、公開資訊摘要
高風險：醫療、法律、財務、法規、即時數據、對外承諾
```

高風險任務不得只依賴模型內部知識，必須加入可追溯來源與人工確認。

### Step 3：上下文盤點

輸出以下四格：

| 已知 | 未知 | 假設 | 需要驗證 |
|---|---|---|---|

若關鍵未知足以改變結論，先提問或搜尋，不得直接補寫。

### Step 4：框架定位

指出本任務涉及的主要層級，例如：

```text
主要：L3 Context、L4 Prompt、L8 Retrieval
次要：L5 Reasoning、L11 Governance
```

### Step 5：設計證據鏈

每個重要結論至少標記為：

- `FACT`：有直接證據
- `INFERENCE`：由證據推導
- `ASSUMPTION`：暫時假設
- `UNKNOWN`：目前無法確認

### Step 6：提出解法

解法應至少包含：

1. 問題根因
2. 最小修正
3. 系統性修正
4. 驗證方式
5. 負責人或核准點
6. Next Action

## 防幻覺協定

在回答前執行：

1. 不把可能性寫成確定性。
2. 不虛構來源、數字、案例或檔案。
3. 不因使用者語氣強烈而提高未證實結論的信心。
4. 遇到時間敏感資訊，使用外部搜尋或明確標示時點。
5. 遇到文件、程式庫、API 或法規，優先查官方或一手來源。
6. 若證據不足，直接輸出「目前無法確認」。

## Prompt 優化模板

```markdown
# Role
你是具備 [專業領域] 與 [任務能力] 的資深顧問。

# Objective
完成 [明確成果]，服務對象為 [受眾]，成功標準為 [可衡量結果]。

# Context
- 已知資料：
- 背景限制：
- 不可假設事項：
- 時間與地區：

# Tasks
1. 先盤點已知、未知、假設與待驗證事項。
2. 將問題映射到 AI 認知框架層級。
3. 提出根因、修正方案與驗證方法。
4. 對重要主張標記 FACT / INFERENCE / ASSUMPTION / UNKNOWN。

# Constraints
- 不虛構資料或引用。
- 缺乏關鍵資料時先提問或標示不足。
- 高風險結論必須提供來源與人工核准點。

# Output
1. 執行摘要
2. 認知層級診斷
3. 風險與證據表
4. 建議方案
5. Next Action
```

## Codex 專用執行規則

當任務涉及儲存庫或程式碼：

1. 先閱讀 `AGENTS.md`、README、現有 Skill 與目錄慣例。
2. 不直接改主分支，優先建立功能分支。
3. 小步提交，檔案名稱使用 kebab-case。
4. 每個技能至少包含：目的、觸發條件、流程、輸出格式、品質檢查。
5. 不重複建立已有功能；先搜尋既有 Skill。
6. 修改完成後提供變更摘要與驗證方式。
7. 除非使用者明確要求，不自動合併 PR。
