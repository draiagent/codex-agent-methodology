## 一、框架一：反向提問法

目的：避免 AI 在資訊不足時直接腦補，先找出模糊處、缺失資料與錯誤假設。

使用時機：

- 大型企劃
- 論文或報告
- 商業模式分析
- 課程設計
- 短影音系列規劃
- 品牌定位

輸出重點：

1. 目前需求判讀
2. 資訊真空地帶
3. 潛在假設與影響
4. 3-5 個關鍵反向提問

啟動語：

```text
使用框架一：反向提問法，先幫我釐清需求。
```

## 二、框架二：防討好五人 AI 智囊團

目的：避免 AI 只附和使用者，改用多視角批判方式替提案排雷。

五位顧問：

| 顧問 | 核心任務 |
|---|---|
| 反駁者 | 找失敗點、過度樂觀假設與低估風險 |
| 本質追問者 | 檢查真需求、偽需求或自嗨點 |
| 擴張者 | 找替代市場、商業模式與跨界機會 |
| 外行人 | 從一般用戶角度檢查是否聽得懂、願意買 |
| 無情的執行者 | 檢查能否落地、明天第一步是什麼 |

主席總結：

- 值不值得做
- 核心修正
- 最大風險
- 缺乏的驗證證據
- Next Action

啟動語：

```text
使用框架二：防討好五人 AI 智囊團，幫我壓力測試這個商模。
```

## 三、與行銷寫作方法論整合

建議工作流：

```text
反向提問法
-> 五人智囊團壓力測試
-> Human Value Funnel
-> 生命價值轉化寫作系統
-> 短影音腳本 / 銷售頁 / 課程文案 / 品牌內容
```

短影音前置使用方式：

1. 先用反向提問法釐清：受眾、痛點、目標、平台、影片長度、CTA。
2. 再用五人智囊團檢查：定位是否過度樂觀、觀眾是否聽得懂、第一步是否可執行。
3. 最後才用行銷寫作方法論產出腳本。
